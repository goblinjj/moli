# 魔力宝贝生产系成本与售卖价格计算器 - 设计文档

## 概述

为魔力宝贝游戏打造一个生产系成本计算器和售卖价格计算器。用户可以查看各制造系物品的制造成本（材料 + 补魔），并通过加价百分比计算售卖价格。所有成本相关参数均可自定义。

## 技术栈

- **Astro** + **React** 组件（交互部分）
- **TypeScript**
- **Tailwind CSS**
- 部署到 **Cloudflare Pages**

## 范围

先实现**造弓**和**料理**两个制造系，架构支持后续扩展更多制造系。

数据来源：https://cgs.hk/produce4.htm（造弓）、https://cgs.hk/produce17.htm（料理）

## 数据模型

### Recipe（配方）

```ts
interface Recipe {
  id: string;              // "bow-1a"
  name: string;            // "輕型弓"
  category: "bow" | "cooking";
  level: number;           // 1-15
  variant: string;         // "A", "B", "C"
  image: string;           // 图片文件名
  materials: Material[];
  mpCost: number;          // 默认耗魔量
}

interface Material {
  name: string;            // "銅條"
  quantity: number;        // 3
  image: string;           // 材料图片文件名
}
```

### PriceConfig（价格配置）

```ts
interface PriceConfig {
  materialPrices: Record<string, number>;  // 材料名 → 单价
  mpPrice: number;                          // 每点魔力价格，默认 1
  recipeMpCosts: Record<string, number>;   // 配方 ID → 自定义耗魔量
  markupRates: {                            // 加价百分比
    bow: number;
    cooking: number;
  };
}
```

## 价格默认规则

- 普通材料：1级 1元/个，2级 2元/个，以此类推
- 矿条：1级 20元/条，2级 40元/条（1矿条 = 20矿石）
- 耗魔量：1级 20点，2级 40点，以此类推
- 补魔单价：每点魔力 1 元
- 所有成本相关参数均可用户自定义

## 核心计算逻辑

### 成本计算（递归）

```
单品成本 = 材料成本总和 + 补魔成本

材料成本 = Σ(材料单价 × 数量)
  - 材料是本系产物 → 材料单价 = 该产物的递归成本
  - 材料是基础材料 → 材料单价 = 用户设定价格

补魔成本 = 耗魔量 × 每点魔力价格
```

递归示例：
```
炒麵麵包(6级) 成本 =
  法國麵包(2级) 成本 × 1 + 炒麵(3级) 成本 × 1 + 补魔(120点)

  法國麵包 成本 = 小麥粉×20 + 牛奶×20 + 鹽×20 + 补魔(40点)
  炒麵 成本 = 小麥粉×20 + 鹽×10 + 海苔×20 + 薑×12 + 补魔(60点)
```

循环依赖保护：递归时记录调用栈，检测到循环则回退到用户设定的材料价格。

### 售卖价格

```
售卖价 = 单品成本 × (1 + 加价百分比 / 100)
```

## 界面设计

现代 Dashboard 风格，卡片式布局。

```
┌──────────────────────────────────────────────────┐
```

### 交互特性

- **制造系切换**：顶部 Tab 切换造弓/料理
- **搜索过滤**：按名称搜索 + 按等级筛选
- **卡片概览**：物品图片、名称、等级、成本、售价
- **展开详情**：材料明细（含图片）、每项费用、补魔费用、成本合计、售卖价
- **左侧设置面板**：实时修改所有成本参数，卡片自动重算

## localStorage 持久化

用户修改即存储，刷新自动恢复：

- 所有材料单价
- 补魔单价
- 每个配方的自定义耗魔量
- 各制造系的加价百分比
- 所有其他成本相关的自定义项

## 图片资源

- 从 cgs.hk 下载到本地 `src/assets/items/`
- 图片 URL 规则：`https://ig2.cgs.hk/g/{hash前2位}/{hash}.png`
- 自包含，不依赖外部服务器

## 扩展性

- 新增制造系只需：添加 JSON 数据文件 + 下载图片 + 在 Tab 中注册
- 数据结构统一，计算逻辑复用

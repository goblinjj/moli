# 补给估价计算器设计文档

## 概述

为魔力宝贝生产计算器新增"补给估价"工具，计算料理和血瓶的市场参考价格及血魔池性价比。同时将站点重构为两大分区：生产系资料（现有功能）和工具（新功能）。

## 需求

### 价格计算
- 料理/血瓶的单价 = 回复值 × 比例（默认 1.5，用户可按条目自定义）
- 一组 = 单价 × 5
- 一箱 = 单价 × 50（10组 × 5个）

### 血魔池计算
- 投入料理/血瓶可增加血魔池容量，按等级打折：
  - 1级 45%, 2级 50%, 3级 55%, 4级 60%, 5级 65%, 6级 70%, 7级 75%, 8级 80%, 9级 85%, 10级 90%
- 血魔池每点成本 = 单价 ÷ (回复值 × 折扣比例)
- 折扣等级直接取料理/血瓶自身等级

### 数据范围
- 仅展示 1-10 级且有回复值（recoveryValue）的料理和血瓶

## 导航架构

### 顶层导航（新增）
```
[ 生产系资料 ] [ 工具 ]
```
- 默认选中"生产系资料"
- "生产系资料"下为现有全部功能（武器/防具/补给/其他/寵物/寶石）
- "工具"下展示工具子导航，每个工具一个 Tab，未来可扩展

### 工具子导航
```
[ 补给估价 ] [ 未来工具... ]
```

## 补给估价页面设计

### 子标签切换
```
[ 料理 ] [ 血瓶 ]
```

### 表格结构

| 图标 | 名称 | 等级 | 回复值 | 比例 | 单价 | 一组(×5) | 一箱(×50) | 血魔池每点成本 |
|------|------|------|--------|------|------|----------|-----------|---------------|
| img | 麵包 | 1 | 100 | [1.5] | 150 | 750 | 7,500 | 3.33 |

- **比例**列为可编辑输入框，默认 1.5
- 所有价格列实时计算
- 按等级排序

## 数据变更

### JSON 数据
在 `cooking.json` 和 `pharmacy.json` 中，为符合条件的条目新增 `recoveryValue` 数值字段：
```json
{
  "id": "cooking-1b",
  "name": "麵包",
  "recoveryValue": 100,
  ...
}
```

### TypeScript 类型
Recipe 接口新增可选字段：
```typescript
recoveryValue?: number;
```

### localStorage 持久化
新增 key `moli-supply-pricing-ratios` 存储自定义比例：
```json
{
  "cooking-1b": 1.5,
  "pharmacy-1a": 1.5
}
```
